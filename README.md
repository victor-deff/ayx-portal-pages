# ayx-portal-pages

## 项目简介

`ayx-portal-pages` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的统一存放与版本管理场所。

## 目录说明

```
ayx-portal-pages/
├── pages/          # 存放所有归档 HTML 页面
├── assets/         # 公共资源文件（CSS、JS、图片等）
└── README.md       # 本文件
```

- **pages/**：每个页面以独立子目录或文件形式存放，命名清晰以便识别。
- **assets/**：存放多个页面共用的样式、脚本或图像资源，减少冗余。

## 页面归档说明

本仓库收录的 HTML 页面具有以下特点：

- 每个页面均为独立完整文件，可脱离外部依赖直接运行（或仅依赖本仓库内资源）。
- 页面内容涉及不同主题，彼此之间无强关联。
- 归档目的包括但不限于：历史记录、预览展示、调试参考、临时发布等。
- 页面可能会根据需要进行更新、替换或移除。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，并确保引用资源路径正确。
- 更新页面：直接覆盖对应文件，建议在提交信息中简要说明变更内容。
- 删除页面：移除对应文件/目录，并注意清理不再使用的资源。
- 资源管理：公共资源尽量统一放在 `assets/` 下，避免重复上传。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务将本仓库部署为站点，直接访问 `pages/` 下的 HTML 文件即可查看对应页面。

本仓库内容遵循各文件自身所附带的许可协议。若无明确说明，则默认仅供学习和参考使用。
